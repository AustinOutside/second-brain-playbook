The Claude Second Brain Playbook

How to turn Claude into a 24/7 second brain with a dual-Mac setup

🧠 The Concept

Claude Cowork (Anthropic's desktop AI assistant) is powerful on its own. But it forgets everything between sessions, can't run tasks while you're away, and only works on the machine in front of you.

The "Second Brain" setup solves all three problems by pairing two MacBooks with a background sync daemon:

Mac 1 — The Second Brain
Always-on at home. Runs scheduled tasks, long jobs, and automations in the background 24/7. Plugged in, lid open, never sleeps.
Mac 2 — Daily Driver
Portable. Interactive work, content creation, research on the go. Same Claude context as Mac 1 — pick up right where you left off.

A background daemon syncs Claude's memory (CLAUDE.md, TASKS.md, memory files) between both machines through iCloud Drive every 60 seconds. Every person Claude knows, every preference, every project — it all follows you.

What This Enables
Start a competitive analysis on your home Mac before bed. Wake up to a finished report. Review it from a coffee shop on your laptop. Add tasks from your phone (via iCloud). Claude becomes an always-available, context-aware assistant that never loses your thread.

🏗 Architecture

The setup has three layers:

1. Claude's Context Files

  • CLAUDE.md — Your identity, preferences, projects, people, glossary, and decisions. Claude reads this at the start of every session.
  • TASKS.md — Shared todo list. Add tasks on either machine.
  • memory/ — Deeper context files: people profiles, project notes, company info, meeting notes.

2. iCloud Drive (Transport Layer)

A shared folder at ~/Library/Mobile Documents/com~apple~CloudDocs/claude-sync/ holds the canonical versions of all synced files. Both Macs read from and write to this folder.

3. The Sync Daemon

A launchd daemon (com.claude.cowork-sync) runs every 60 seconds on both Macs. It finds the newest Cowork session's files and syncs them bidirectionally with iCloud using MD5 hash-based change detection.

What Syncs

  • CLAUDE.md (identity, prefs, context)
  • TASKS.md (shared todo list)
  • memory/ (people, projects, notes)

What Stays Local

  • MCP connector credentials
  • Installed plugins
  • Session history
  • Scheduled tasks (Mac 1 only)
Why Not Symlinks?
Symlinks into iCloud Drive don't work reliably — iCloud treats them as regular files or ignores them. Timestamps aren't preserved between machines either, so file-modification-time comparisons fail. The daemon uses MD5 hashes instead, which are bulletproof.

Key Technical Details

  • Cowork stores CLAUDE.md in per-session folders at ~/Library/Application Support/Claude/local-agent-mode-sessions/. The session UUID changes every conversation.
  • The daemon dynamically finds the newest session directory each cycle.
  • When a new session is detected, iCloud is treated as source of truth (seeds the new session, doesn't overwrite iCloud with the blank default).
  • /bin/bash needs Full Disk Access to write to iCloud Drive (launchd runs bash directly, not through Terminal).

🔄 Setting Up the Sync

Here's the high-level setup process. Ask Claude to help you build and customize the sync script for your specific setup.

Prerequisites

  • Two Macs with iCloud Drive enabled and signed into the same Apple ID
  • Claude Desktop app installed on both
  • /bin/bash added to Full Disk Access on both Macs (System Settings > Privacy & Security > Full Disk Access)

What the Script Does

  • Finds the newest Cowork session directory
  • Computes MD5 hashes of local and iCloud versions of each file
  • Compares hashes against last-known state (stored in ~/.claude-sync/state/)
  • Copies the changed version to the other side
  • Detects new sessions and seeds them from iCloud (source of truth)
  • Logs all activity to ~/.claude-sync/sync.log

Installation Steps

1. Create the sync directory: mkdir -p ~/.claude-sync/state
2. Create the iCloud sync folder: mkdir -p ~/Library/Mobile\ Documents/com~apple~CloudDocs/claude-sync
3. Write the sync script to ~/.claude-sync/cowork-sync.sh (ask Claude to generate it for you)
4. Make it executable: chmod +x ~/.claude-sync/cowork-sync.sh
5. Create a launchd plist at ~/Library/LaunchAgents/com.claude.cowork-sync.plist that runs the script every 60 seconds
6. Load the daemon: launchctl load ~/Library/LaunchAgents/com.claude.cowork-sync.plist
7. Repeat on the second Mac
Pro Tip: Let Claude Build It
Open Cowork and say: "I want to set up a dual-Mac second brain with iCloud sync. Help me create the sync script and launchd daemon." Claude will generate everything customized for your system. This playbook documents the architecture — Claude handles the implementation.

Always-On Mac Configuration

  • Prevent automatic sleeping: System Settings > Battery > Options > "Prevent automatic sleeping when display is off" → ON
  • Disable auto-install updates: System Settings > General > Software Update > Automatic Updates > "Install macOS updates" → OFF (prevents surprise restarts)
  • Keep the lid open (or use an external display for clamshell mode)
  • Display sleep is fine (set to 5 min — the machine stays awake)

📝 Memory System

The more context Claude has, the more it feels like a real colleague. Your CLAUDE.md is Claude's long-term memory — it carries across every session on both machines and compounds over time.

Recommended CLAUDE.md Structure

Top — Daily Context

  • About Me — who you are, what you do
  • Preferences — tone, format, deliverable style
  • Work — company, role, products, team
  • Projects — active projects + status
  • People — contacts with role/context
  • Terms — acronyms and shorthand

Bottom — Infrastructure

  • Capabilities — plugins and skills
  • Sync Setup — paths, commands
  • Scheduled Tasks — what runs automatically
  • Resources — reference docs
  • Decision Log — key choices with dates
  • Lessons Learned — insights for Claude
Just Tell Claude
You never need to edit files manually. Say: "Remember that Sarah is my accountant" or "Add a note that I prefer dark mode." Claude updates CLAUDE.md for you, and the daemon syncs it to the other machine within 60 seconds.

The memory/ Directory

For deeper context, use the memory/ folder. It syncs too:

  • memory/people/ — detailed profiles
  • memory/projects/ — project notes and briefs
  • memory/work/ — company context, team info
  • memory/glossary.md — full terminology reference
The Compounding Effect
After a few weeks, Claude knows your projects, your people, your communication style, and your preferences. It stops asking clarifying questions and starts anticipating what you need. Spend 30 seconds adding context now, save hours over the next month.

Memory Best Practices

  • End-of-session habit: "Save anything important from this session to memory."
  • After meetings: "Save the key points and action items from my call with [person]."
  • When you make a decision: "Log this decision: we're going with X because of Y."
  • When something changes: "Update memory: [person] is no longer at [company], new contact is [name]."

⏰ Scheduled Tasks

Scheduled tasks run on your always-on Mac. They fire automatically — no interaction needed. Think of them as personal automations that work while you sleep, commute, or focus on other things.

Ideas for Scheduled Tasks

Monitoring & Intel

  • Morning briefing (calendar, email digest, action items)
  • Job board scanner for target roles
  • Competitor website change detection
  • Industry news digest
  • Website health checks
  • RSS feed monitoring

Productivity & Reports

  • Daily task recap + tomorrow's priorities
  • Weekly project status roundup
  • Friday "week in review" digest
  • End-of-day summary
  • Email thread digest (unread/flagged)
  • Monthly goal check-in
Creating Tasks
On your always-on Mac, just say: "Create a scheduled task that checks my inbox every morning at 7am and summarizes anything urgent." Claude sets it up with the /schedule skill. You can also create one-time tasks: "Remind me tomorrow at 3pm to follow up with [person]."
Scope Each Task Narrowly
The best scheduled tasks do one thing well. "Scan job boards and rank results" is better than "Do my entire morning routine." Split complex workflows into separate tasks that run in sequence (e.g., 6:00am scan, 6:15am summarize, 6:30am draft responses).

🔗 Connectors

Connectors let Claude read and act on your real data — email, calendar, docs, and more. They use MCP (Model Context Protocol) to bridge Claude and your tools securely. 38+ connectors are available including Gmail, Google Calendar, Notion, Slack, Google Drive, HubSpot, Jira, Salesforce, and more.

Example Prompts That Use Connectors

Email Triage
"Check my Gmail for anything urgent from the last 12 hours. Summarize the top 5 threads I should respond to and draft replies for the straightforward ones."
Meeting Prep
"I have a call with [person] at 2pm. Pull their recent emails, check our calendar invite for context, and search Notion for any prior notes. Give me a one-page prep brief."
Calendar Audit
"Look at my calendar for next week. Flag double-bookings, find my longest free block, and suggest which meetings could be declined."
Discovering New Connectors
Just mention a tool you want to use: "Can you check my Slack messages?" Claude will search the MCP registry, find the connector, and offer to help you install it. You can also ask directly: "What connectors are available for project management?"
Connectors Are Per-Machine
If you connect Gmail on Mac 1, it's not automatically connected on Mac 2. Install and authenticate connectors on each machine separately. Context (CLAUDE.md) syncs, credentials don't.

🎯 Daily Workflows

Morning (Always-On Mac)

1. Open Cowork — your morning briefing is already generated by scheduled tasks
2. Review overnight outputs — "Show me today's briefing"
3. Triage email — "Check Gmail for anything urgent, draft quick replies"
4. Kick off heavy work — "Research [topic], have it ready by noon"
5. Update tasks — "What's on my TASKS.md? Add [new item]"
6. Head out — Mac 1 keeps running your jobs

Midday (Daily Driver)

1. Open Cowork — full context syncs in automatically
2. Quick tasks — "Draft that email" or "Update the project doc"
3. Meeting prep — "I have a call in 30 min with [person], prep me"
4. Edits sync back to Mac 1 automatically

Evening

1. Review the day — "Summarize what I accomplished based on TASKS.md"
2. Save context — "Save anything important from today to memory"
3. Queue overnight work on Mac 1 — "Generate a [deliverable] by morning"
4. Update memory — "Remember that [person] mentioned [detail] today"
The "Leave a Note" Pattern
Before ending a session: "Add a note to TASKS.md that I should follow up on [X] tomorrow." When you open Cowork on the other machine, it's right there. Like a sticky note that syncs across devices.

🛠 Custom Skills

Skills are reusable workflows you create for tasks you repeat. Instead of re-explaining what you want every time, write instructions once and trigger them forever.

When to Create a Skill

  • You find yourself giving Claude the same instructions more than twice
  • You have a specific format that needs to be followed exactly
  • You want a slash command shortcut for a common workflow
Create a Skill
"Create a new skill called 'weekly-review' that checks my TASKS.md, summarizes what I completed this week, flags overdue items, and generates a clean weekly summary."
Capture a Great Workflow as a Skill
"We just did a great workflow for [X]. Turn what we just did into a reusable skill so I can run it again with one command."
Keep Skills Focused
One giant skill performs worse than three focused skills. "draft-linkedin-post," "weekly-seo-check," and "competitor-snapshot" are better than one "do-everything-marketing" skill.

⚡ Power Tips

1. Front-Load Your Memory in Week One
Spend 10 minutes a day adding people, projects, terms, and preferences. By day 7, Claude feels like a colleague who's worked with you for months.
2. TASKS.md Is Your Cross-Device Todo
Add tasks on either machine — they sync. Ask "What's on my plate?" from anywhere. End each session with "Update TASKS.md with what we did and what's next."
3. Ask for Interactive HTML Deliverables
For anything complex, ask Claude for an interactive HTML file — checkboxes, collapsible sections, copy buttons, progress bars. Way more useful than plain text.
4. Chain Tasks Across Time
"Generate 20 blog post ideas" (overnight) → "Take the top 5 and draft outlines" (morning). Your second brain works the night shift.
5. The Decision Log Prevents Re-Debates
Log key decisions. Future sessions won't re-ask settled questions. "We chose Squarespace because..." → saved, never revisited unless you want to.
6. Turn Great Sessions into Skills
When Claude nails a workflow, say "Turn what we just did into a reusable skill." Next time, one command recreates the whole thing.
7. Use Sub-Agents for Parallel Work
Claude can split work across sub-agents. "Research these three competitors in parallel" → ~10 min instead of ~30 min. Great for batch research and multi-file analysis.
8. Use /compact for Long Sessions
When sessions get long, type /compact to compress conversation history while preserving key details. Essential for marathon sessions.
9. "Save to Memory" as Your Session Closer
Before ending any productive session: "Save anything important from this session to memory." Claude extracts key decisions, contacts, and updates — so the next session picks up where you left off.
10. Scheduled Tasks Can Use Your Connectors
Tasks on your always-on Mac can access Gmail, Calendar, Notion (if connected). Your morning briefing can pull from real email and calendar data, not just guesses.
11. Install Plugins and Connectors on BOTH Machines
Context syncs, tools don't. If you add a plugin on one Mac, install it on the other too. Same for MCP connectors — authenticate each one per-machine.

🔧 Troubleshooting

New session missing my context
Normal — daemon seeds within 60 seconds. Force sync: bash ~/.claude-sync/cowork-sync.sh
"Operation not permitted" in error log
/bin/bash needs Full Disk Access. System Settings > Privacy & Security > Full Disk Access > + > Cmd+Shift+G > /bin > select bash > Open > toggle ON. Restart daemon.
Daemon stopped running
launchctl list | grep cowork-sync — if empty, reload: launchctl load ~/Library/LaunchAgents/com.claude.cowork-sync.plist
Always-on Mac went to sleep
Battery > Options > "Prevent automatic sleeping" must be ON. Keep lid open. No retry for missed scheduled tasks.
Scheduled task didn't run
Tasks only run while the Mac is awake and Claude Desktop is open. Check: lid open? Sleep prevention on? Claude app running? Was there an auto-update restart?
Changes not syncing between machines
Check logs on both: tail -10 ~/.claude-sync/sync.log and cat ~/.claude-sync/sync-stderr.log. Most common fix: add /bin/bash to Full Disk Access.
Nuclear reset
Stop daemon, delete state, restart: launchctl unload ~/Library/LaunchAgents/com.claude.cowork-sync.plist && rm -rf ~/.claude-sync/state/ && launchctl load ~/Library/LaunchAgents/com.claude.cowork-sync.plist
